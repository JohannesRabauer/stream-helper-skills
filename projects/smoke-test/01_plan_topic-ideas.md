# Topic ideas

## Strategy: pragmatic
**Selected: yes**

**Title:** Building a Local-First AI Assistant with Spring Boot and Ollama

**Why it matters now:** Teams want to ship AI features without committing to a cloud
API bill or sending user data off-box. A live, working local-model integration is a
concrete answer to "can we actually do this ourselves?"

**Expected audience:** Java/Spring backend developers evaluating AI features for
their own products, who want a real implementation over another slide deck.

**Concrete coding deliverable:** A small Spring Boot service that swaps between
Ollama and OpenAI behind one interface, with a working chat endpoint by the end of
the stream.

## Strategy: beginner-friendly
**Selected: no**

**Title:** Your First AI-Powered Java App: A Beginner's Walkthrough

**Why it matters now:** AI tooling feels intimidating from the outside, especially
for developers who've only used Java for "boring" CRUD apps. Seeing it built from a
blank project lowers the barrier to entry.

**Expected audience:** Junior-to-mid Java developers who've heard about AI APIs but
never wired one up themselves.

**Concrete coding deliverable:** A minimal Spring Boot app that calls an LLM API and
returns a response in a REST endpoint, built live from `spring init`.

## Strategy: hype
**Selected: no**

**Title:** I Gave My Java App a Brain (and It Actually Works)

**Why it matters now:** "AI in Java" content is dominated by Python-first framing;
a confident, high-energy Java-native take stands out and travels well on socials.

**Expected audience:** Broader dev-Twitter/YouTube audience who click on bold claims
and stay for a real demo.

**Concrete coding deliverable:** A live demo app that takes a user prompt and
produces a working code suggestion, shipped and running by the end of the stream.
